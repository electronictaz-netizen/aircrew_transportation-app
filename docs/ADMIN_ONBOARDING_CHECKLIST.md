# Company Onboarding Checklist for Administrators

Use this checklist when onboarding a new company to ensure they're set up correctly.

## Pre-Onboarding (Your Tasks)

- [ ] Create the company in Admin Dashboard
- [ ] Set company name and subdomain
- [ ] Configure subscription tier (Free, Basic, or Premium)
- [ ] Set subscription status to "Active"
- [ ] Create initial admin user account
- [ ] Send invitation email to company administrator

## Initial Setup (Company Administrator Tasks)

### Company Configuration

- [ ] Company administrator logs in
- [ ] Verify company information is correct
- [ ] Review subscription tier and status
- [ ] Update company settings if needed (Company Settings)

### User Management

- [ ] Add all drivers to the system
  - [ ] Driver name
  - [ ] Email address
  - [ ] Phone number
  - [ ] License number
  - [ ] Notification preferences (Email or Both)
- [ ] Invite managers (if applicable)
- [ ] Verify all users can log in successfully

### Location Setup

- [ ] Add frequently used pickup locations
- [ ] Add frequently used dropoff locations
- [ ] Verify locations are saved correctly
- [ ] Test location selection when creating trips

### First Trip Creation

- [ ] Create a test trip
- [ ] Verify all fields are working
- [ ] Assign trip to a driver
- [ ] Verify driver receives notification
- [ ] Test trip editing functionality
- [ ] Test trip deletion (if needed)

## Training & Documentation

- [ ] Share `COMPANY_ONBOARDING_GUIDE.md` with company
- [ ] Share `COMPANY_QUICK_START.md` with company
- [ ] Provide application URL
- [ ] Explain user roles (Admin, Manager, Driver)
- [ ] Demonstrate key features:
  - [ ] Creating trips
  - [ ] Assigning drivers
  - [ ] Managing locations
  - [ ] Checking flight status
  - [ ] Daily assignment emails

## Verification

- [ ] All users can log in
- [ ] All drivers are added and active
- [ ] Locations are saved
- [ ] Test trip created successfully
- [ ] Notifications are working
- [ ] Company administrator understands the system
- [ ] Support contact information provided

## Post-Onboarding

- [ ] Schedule follow-up (1 week after onboarding)
- [ ] Check for any issues or questions
- [ ] Verify company is actively using the system
- [ ] Collect feedback for improvements

## Support Information to Provide

- Application URL: `[Your App URL]`
- Support Email: `[Your Support Email]`
- Documentation: `COMPANY_ONBOARDING_GUIDE.md` and `COMPANY_QUICK_START.md`
- Emergency Contact: `[Emergency Contact]`

## Notes

- **Subscription Tiers**: 
  - Free: Basic features
  - Basic: Standard features
  - Premium: All features including advanced reporting

- **User Roles**:
  - Admin: Full access, can manage company settings
  - Manager: Can manage trips and drivers, cannot manage company settings
  - Driver: View-only access to assigned trips

- **Important Features to Highlight**:
  - Bulk trip assignment
  - Saved locations
  - Flight status checking
  - Daily assignment summaries
  - Mobile app installation (PWA)
